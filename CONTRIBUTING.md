# Contributing

## Branching

- Create feature branches from `main`.
- Use clear branch names, for example `service/lts-002-equipment-induction`.

## Pull Requests

- Keep PRs focused to one change set where possible.
- Include context, impacted services, and review notes.
- Request review from service owner and repository maintainer.

## Service Entry Checklist

- Unique `id` assigned
- `owner` and contact details confirmed
- Request method and link validated
- Target response and fulfilment times set
- Review cadence and date updated
